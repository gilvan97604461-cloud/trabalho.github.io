Fundamentação Histórica

Linha do Tempo: 70 anos de linguagens

1957–1960 FORTRAN (IBM, John Backus) · COBOL (Grace Hopper) · LISP — primeiras linguagens de alto nível
1972 C (Dennis Ritchie, Bell Labs) — base de todo o software moderno de sistemas + Unix
1985–1991 C++ (OOP) · Python (Guido van Rossum) — produtividade e orientação a objetos
1995 Java (Sun, "write once run anywhere") · JavaScript (Brendan Eich, 10 dias!) · PHP — explosão da web
2009–2015 Go (Google, concorrência simples) · Rust (Mozilla, segurança sem GC) — era do cloud e sistemas seguros

Os 5 Grandes Paradigmas

📋 Procedural / Imperativo

Sequência de instruções que modificam estado. Unidade: rotina/função. Exemplos: FORTRAN, COBOL, C

🧱 Orientado a Objetos

Objetos com dados e comportamentos. Encapsulamento, herança, polimorfismo. Exemplos: Java, Python, C++

λ Funcional

Funções puras, imutabilidade, sem efeitos colaterais. Recursão no lugar de laços. Exemplos: LISP, Haskell, Scala

🔗 Lógico

Fatos e regras de inferência lógica. Dedução automática de soluções. Exemplo: Prolog (IA simbólica)

📜
Script / Dinâmico

Tipagem dinâmica, interpretado, alta produtividade. Trade-off: performance. Exemplos: Python, Ruby, JavaScript — dominam automação e IA

Tecnologia & Sociedade

Estudo de Caso: Do COBOL ao Cloud

🏦 O desafio bancário

95% das transações em ATMs e 80% das operações com cartão de crédito no mundo ainda rodam em COBOL. Bradesco, Itaú e Banco do Brasil têm mainframes IBM com milhões de linhas de COBOL acumuladas há décadas.

🔄 Migração no Brasil

Uma cooperativa financeira nacional migrou de COBOL para Java com apoio de consultoria. Caso global: ~1.000.000 linhas de COBOL convertidas, 400+ tabelas Oracle, anos de trabalho especializado.

Por que não migrar tudo de uma vez?
⚠️
Confiabilidade: COBOL processou bilhões de transações por décadas sem falhar
💰
Custo: Reescrita total de sistemas críticos é extremamente cara e arriscada
👨‍💻
Escassez de talento: Poucos programadores COBOL + falta de contexto de negócio nos novos devs
📋
Regulação: Bancos regulados — qualquer mudança requer validação extensiva pelo Bacen
💡 Mercado dual: alta demanda por COBOL seniores (salários altos pela escassez) + alta demanda por arquitetos de modernização

LGPD & Marco Civil da Internet

⚖️
Marco Civil da Internet
Lei nº 12.965 · 2014
  • ✓ Princípios e direitos para uso da internet
  • ✓ Responsabilidade de provedores por conteúdo de terceiros
  • ✓ Neutralidade de rede
  • ✓ Base para proteção digital no Brasil
🛡️
LGPD
Lei nº 13.709 · 2018
  • ✓ Consentimento explícito para coleta de dados
  • ✓ Direito de exclusão e portabilidade de dados
  • ✓ Notificação obrigatória de vazamentos
  • ✓ Inspirada no GDPR europeu
🔧 Impacto técnico nas linguagens e sistemas

A LGPD impôs mudanças concretas: criptografia de dados sensíveis, logs de auditoria, APIs de exportação/exclusão, anonimização. Isso aumentou a demanda por Java e Python no mercado corporativo — linguagens com ecossistemas maduros de segurança e conformidade.

Análise Crítica

Trade-offs, Riscos e Tendências

⚡ Trade-offs Históricos
Performance vs. Segurança: C entrega desempenho máximo mas gerou décadas de vulnerabilidades (buffer overflow, etc.)
Produtividade vs. Performance: Python é até 20× mais lento que C — mas suas bibliotecas (NumPy, TensorFlow) são escritas em C por baixo
Portabilidade vs. Controle: JVM do Java promete "write once run anywhere" ao custo de overhead de memória
⚠️ Riscos Emergentes
🎯 Viés algorítmico: Sistemas de crédito e reconhecimento facial podem amplificar desigualdades raciais no Brasil
🔒 Supply chain: Incidente log4j (2021) — uma lib Java comprometeu milhares de sistemas globais
☁️ Vendor lock-in: Python "open source" rodando no SageMaker da AWS não é verdadeiramente portável
✅ Tendências sólidas
Rust para sistemas críticos · Go para microsserviços · Python para IA/ML · TypeScript sobre JS
⚡ Hype com cautela
"Qualquer linguagem faz tudo" · "COBOL vai morrer" (errado desde 1980!) · Nova linguagem como solução completa
